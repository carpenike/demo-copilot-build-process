# Bootstrap Configuration Template

> **Produced by:** @5-deployment agent
> **Consumed by:** `scripts/check-prerequisites.sh --from-config <path>`

The `bootstrap.conf` file is a shell-sourceable config that tells the bootstrap
script which Azure services the project needs. It is generated by the
@5-deployment agent based on the project's ADRs and infrastructure design.

## File Location

```
projects/<project>/infrastructure/bootstrap.conf
```

## Usage

```bash
# Check prerequisites using the config
./scripts/check-prerequisites.sh my-project dev \
    --from-config projects/my-project/infrastructure/bootstrap.conf

# Check + auto-create missing resources
./scripts/check-prerequisites.sh my-project dev \
    --from-config projects/my-project/infrastructure/bootstrap.conf --fix
```

## Template

```bash
# bootstrap.conf — Auto-generated by @5-deployment agent
# Defines which Azure services this project requires.
# Sourced by: scripts/check-prerequisites.sh --from-config <this-file>
#
# DO NOT put secrets in this file — it is committed to the repo.
# Secrets are generated by the bootstrap script and stored in GitHub Secrets.

# ─── Feature Flags ──────────────────────────────────────────────────────────
# Set to "true" to enable checks/creation of these Azure services.
# Leave empty or remove the line to skip.

# Azure OpenAI Service — set if project uses LLMs or embeddings
WITH_OPENAI=""

# Azure AI Search — set if project uses vector/hybrid search (RAG)
WITH_SEARCH=""

# Entra ID App Registration — set if project has user-facing authentication
WITH_ENTRA_AUTH=""

# ─── Entra ID App Roles (requires WITH_ENTRA_AUTH=true) ─────────────────────
# JSON array of app roles to create. Leave empty for no roles.
# Example for a chatbot with Employee and Administrator roles:
# APP_ROLES='[
#     {"allowedMemberTypes":["User"],"displayName":"Employee","isEnabled":true,"value":"Employee"},
#     {"allowedMemberTypes":["User"],"displayName":"Administrator","isEnabled":true,"value":"Administrator"}
# ]'
APP_ROLES=''

# ─── Extra GitHub Secrets ───────────────────────────────────────────────────
# Space-separated list of additional GitHub repository secrets the project
# needs beyond the core set (AZURE_CLIENT_ID, AZURE_TENANT_ID, etc.).
# The bootstrap script will check these exist and prompt to set them.
#
# Common extras:
#   POSTGRES_ADMIN_PASSWORD  — database admin password
#   REDIS_ACCESS_KEY         — Redis cache access key
#   SERVICENOW_INSTANCE_URL  — ServiceNow integration URL
EXTRA_SECRETS=""

# ─── Location Overrides ────────────────────────────────────────────────────
# Some Azure services may need a different region than the resource group.
# For example, PostgreSQL Flexible Server may not be available in all regions.
# Set this to override the database module's location parameter.
DATABASE_LOCATION=""
```

## How the @5-deployment Agent Generates This

The agent should:
1. Read the project's ADRs to determine which Azure services are used
2. Read `projects/<project>/src/app/config.py` to find all Settings fields
3. Map Settings fields to required secrets
4. Generate `bootstrap.conf` with the appropriate flags set

### Decision Matrix

| If the project uses... | Set these flags |
|------------------------|-----------------|
| Azure OpenAI / GPT / embeddings | `WITH_OPENAI=true` |
| Azure AI Search / vector search | `WITH_SEARCH=true` |
| Entra ID / user authentication | `WITH_ENTRA_AUTH=true` |
| PostgreSQL | `EXTRA_SECRETS` includes `POSTGRES_ADMIN_PASSWORD` |
| Redis | `EXTRA_SECRETS` includes `REDIS_ACCESS_KEY` |
| Any external API requiring keys | Add to `EXTRA_SECRETS` |
