## Azure Resources (must exist before first deployment)

### Resource Groups
| Environment | Resource Group Name | CI Principal Role |
|-------------|--------------------|--------------------|
| Dev | `rg-policy-chatbot-dev` | **Owner** (for RBAC assignments) |
| Staging | `rg-policy-chatbot-staging` | **Owner** |
| Production | `rg-policy-chatbot-prod` | **Owner** |

> The CI service principal MUST have **Owner** (not Contributor) on each
> resource group so Bicep can create role assignments (AcrPull, Key Vault
> Secrets User, AI Search roles, OpenAI roles, Blob Storage roles).

### Azure Container Registry (shared across projects)
- **ACR Name:** `<your-acr-name>` (globally unique)
- **SKU:** Basic (dev) / Standard (production)
- Must be created before the first CI build

### Azure Services Provisioned by Bicep
These are created automatically by `main.bicep` — no manual setup needed:
- Azure Container Apps managed environment + container app
- Azure Database for PostgreSQL Flexible Server
- Azure Cache for Redis
- Azure Blob Storage (with `policy-docs` container)
- Azure AI Search (Basic SKU, AAD+ApiKey auth)
- Azure OpenAI Service (GPT-4o + text-embedding-ada-002 deployments)
- Azure Key Vault
- Application Insights + Log Analytics Workspace

### Microsoft Entra ID App Registration (manual)
1. Register an application in Microsoft Entra ID
2. Configure the following:
   - **Redirect URIs:** Add web platform redirect for each environment
   - **App roles:** Create two app roles:
     - `Employee` — assigned to all users
     - `Administrator` — assigned to HR admins
   - **API permissions:** `Microsoft Graph → User.Read` (delegated)
   - **Token configuration:** Set `requestedAccessTokenVersion: 2`
3. Note the **Application (client) ID** and **Directory (tenant) ID**
4. Store as GitHub secrets (see below)

## Service Principal Setup

Create a service principal with federated credentials for GitHub Actions
(OIDC — no stored secrets):

```bash
# Create the service principal
az ad sp create-for-rbac --name "sp-policy-chatbot-ci" --role Owner \
    --scopes /subscriptions/<sub-id>/resourceGroups/rg-policy-chatbot-dev \
             /subscriptions/<sub-id>/resourceGroups/rg-policy-chatbot-staging \
             /subscriptions/<sub-id>/resourceGroups/rg-policy-chatbot-prod

# Configure federated credentials for GitHub Actions OIDC
az ad app federated-credential create --id <app-object-id> \
    --parameters '{"name":"github-main","issuer":"https://token.actions.githubusercontent.com","subject":"repo:<org>/<repo>:ref:refs/heads/main","audiences":["api://AzureADTokenExchange"]}'

az ad app federated-credential create --id <app-object-id> \
    --parameters '{"name":"github-pr","issuer":"https://token.actions.githubusercontent.com","subject":"repo:<org>/<repo>:pull_request","audiences":["api://AzureADTokenExchange"]}'

az ad app federated-credential create --id <app-object-id> \
    --parameters '{"name":"github-env-dev","issuer":"https://token.actions.githubusercontent.com","subject":"repo:<org>/<repo>:environment:dev","audiences":["api://AzureADTokenExchange"]}'

az ad app federated-credential create --id <app-object-id> \
    --parameters '{"name":"github-env-staging","issuer":"https://token.actions.githubusercontent.com","subject":"repo:<org>/<repo>:environment:staging","audiences":["api://AzureADTokenExchange"]}'

az ad app federated-credential create --id <app-object-id> \
    --parameters '{"name":"github-env-production","issuer":"https://token.actions.githubusercontent.com","subject":"repo:<org>/<repo>:environment:production","audiences":["api://AzureADTokenExchange"]}'
```

## First Deployment Notes

1. **RBAC propagation:** On the first deploy, Azure RBAC role assignments
   (AcrPull, Key Vault Secrets User, AI Search roles, OpenAI roles) take
   1–5 minutes to propagate. The CI pipeline handles this with a retry pattern.

2. **Database migrations:** Alembic migrations run automatically on container
   startup via `entrypoint.sh`. No manual migration step is needed.

3. **AI Search index:** The application creates the `policy-chunks` index on
   first startup (via the lifespan handler). The `Search Service Contributor`
   RBAC role enables this.

4. **OpenAI model deployments:** GPT-4o and text-embedding-ada-002 are
   provisioned by Bicep. Model availability varies by region — if deployment
   fails, try a different Azure region.
