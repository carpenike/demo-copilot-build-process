# `docs/runbooks/`

Service-specific runbooks are produced by the Monitor Agent and stored
per-project at `projects/<project>/operations/runbook.md`.

This directory holds **cross-cutting** operational documentation:
- Platform-wide incident response procedures
- Shared escalation paths
- Infrastructure runbooks (AKS, Azure Database for PostgreSQL, networking)

If no cross-cutting runbooks exist yet, this directory will be empty.
